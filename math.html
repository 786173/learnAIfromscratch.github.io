Intro

This is a (ongoing) workshop to learn the math needed for various prereqs in university AI intro courses, and is Part IB of the so-called AI Tripos to get to a level of competency to satisfy prerequisites in future AI courses. No background is assumed.

README

This is experimental. In a regular university curriculum you would likely take 3 semesters of calculus (essentially the entire calculus book written by Stewart). You would take a linear algebra class, an intro probability class, a class about linear regression/intro to stats. You would take an 'intro to proofs' style course being flooded with advanced strategies, and then yet another proof based class in discrete math covering an enormous grab bag of topics. Every one of these classes you would have TAs/office hours/recitations and peer study groups to help you.

We don't have any of these resources to help us, or the motivation to complete them in the beginning. So we need to do things differently, it's just us by ourselves trying to figure out this material. University you have a specific goal: graduate and not lose all your tuition money, and you work towards that single goal of graduation and drives you to complete the material. You're given a strict schedule to follow until you graduate. However doing this yourself, there is no schedule, and ample opportunity to just quit at anytime because quitting is consequence free for us. So that's what most people do.

Instead we are going to learn basic math by proving simple facts about it at the same time. This way we are doing conceptually simple abstractions incrementally to more difficult abstractions and not being bombarded with a large number of advanced topics while expected to learn how a proof works at the same time. We're also going to learn things like polynomials from the lens of linear algebra from the start. We're also just going to backfill whatever we need on the way.

A few resources won't have any provided solutions, we'll be expected to figure it out ourselves. This is actually how you learn. In definition-theorem-proof style books the author will show you dozens of proofs per chapter, and then expect you to mimic what they did in order to solve the exercises. In a way you already have the answers in each chapter and you'll see what this means when we start Terence Tao's Analysis I, and his Linear Algebra notes.

Because all this is experimental it could fail, but you'll still know more about math than if you spent the same time in a massive precalculus book.

Arithmetic with natural numbers

  • YouTube - Arithmetic and Geometry Math Foundations 2

An interesting explanation of multiplication.

  • Laws for addition
    • n + m = m + n (Commutative)
      • Changing the order of the elements.
    • (k + n) + m = k + (m + n) (Associative)
      • Changing the grouping of the elements.
    • n + 1 = s(n) (Successor)
  • Laws for multiplication
    • nm = mn (Commutative)
    • (kn)m = k(nm) (Associative)
    • n \(\cdot\) 1 = n (Identity)
  • Note the different notation for multiplication:
    • nm
    • n \(\cdot\) m
    • n \(\times\) m
  • Distributive laws
    • k(n + m) = kn + km
    • (k + n)m = km + nm

Exercise: Prove why the multiplication and distributive laws work. Since we have no idea what a proof is yet, we're just going to walk through some examples.

Identity law: n \(\cdot\) 1 = n.

  • We are creating n groups of 1. According to our definition of multiplication |||| \(\cdot\) | would be 4 groups of | or ||||. This is still 4.

Commutative law: nm = mn.

  • ||| \(\cdot\) || = || || || or 6. Rearranging this, || \(\cdot\) ||| is ||| ||| which is again 6.

Associative law: (kn)m = k(nm).

  • If k = |, and n = ||, and m = ||| and parens expressions are evaluated first:
    • Left side: (kn)m. We first evaluate (kn) as || (2 groups of 1, the identity law), and 2m is ||| ||| (2 groups of 3) for a total of 6.
    • Right side: k(nm): In the parens nm is 2 groups of 3, so ||| |||. k(6) is the identity law, one group of 6. Thus (kn)m = k(nm).

Distributive law: (k + n)m = km + nm.

  • If k = ||||, and n = ||, and m = |:
    • Left side: (|||| + ||) is 6, and (6) x 1 is six groups of 1 or 6 according to the identity law.
    • Right side: km + nm: 4 groups of one is 4, plus two groups of 1 is 2 (identity law). Answer is again 6.

2.1 The Peano Axioms

  • The five Axioms
    • Axiom 2.1: 0 is a natural number
    • Axiom 2.2: If n is a natural number, then n++ is also a natural number
    • Axiom 2.3: 0 is not the successor of any natural number
    • Axiom 2.4: Different natural numbers must have different successors
    • Axiom 2.5: Let P(n) be any property pertaining to a natural number n. Suppose that P(0) is true, and suppose that whenever P(n) is true, P(n++) is also true. Then P(n) is true for every natural number n.
      • What this translates to is: We make an assumption about a property P(n) being true, for example you are looking at a sequence of numbers, and you figure out that there is a pattern where you are able to deduce the next number in the sequence. You call your hypothesis P(n), and you assume this is true meaning you do not show P(n), you show P(0) and P(n+1) are true, and both these form a logical implication meaning that if they are both true, then your initial assumption of P(n) being true holds.

Remark 2.1.10 and the template in Proposition 2.1.11 is probably the clearest definition of induction I've seen. P means 'property'. Note the example where no natural number can be 'a half number', because of the increment step n++. The base case of 0 (as per Axiom 2.5, and Axiom 2.1) fails if we are using 'half numbers' instead of natural numbers because if zero is a natural number, and any n++ is another natural number, the only next successive number is 1 it can't be 0.5 or -3 or 3/4. The way you should view these axioms about the natural numbers is:

  • 0 is an element of N
  • if x is an element of N, then so is x + 1.
  • nothing else is an element of N

Aside: Induction and Recursion

Induction and recursion are the same concept. As an example, the specification of a program function twoExp(n) that when given a value n, returns \(2^n\). The examples you write for this program are as follows:

  • twoExp(0) = 1
  • twoExp(1) = 2
  • twoExp(2) = 4
  • twoExp(3) = 8
  • twoExp(4) = 16
  • twoExp(5) = 32

You notice the pattern: the result of the next value, is 2x the previous value. twoExp(5) is 2 * twoExp(4). You write an algorithm using the above examples: if input = 0, then return 1, otherwise return 2 * twoExp(n-1). You make an assumption, that the input will always be non-negative. Walking through the program execution, twoExp(3) evaluates:

  • 2 * twoExp(3-1)
  • 2 * (2 * twoExp(2-1))
  • 2 * (2 * (2 * twoExp(1-1)))
  • 2 * (2 * (2 * (1))) = \(2^3\)

To prove this by induction on n, we try the base case P(0). If n = 0 then twoExp(0) evaluates to 1, which is \(2^0\). What is P(n)? According to our recursive definition, n = 2*P where the value of P = twoExp(n-1) or \(2^{n-1}\). Our inductive step is n, since our algorithm is inductively defined n-1, and we wish to prove the successor n + 1, which is n. 2*P = \(2*2^{n-1}\) or \(2^n\).

Note that we didn't have to do anything extra to prove this recursive function. That's because an inductive proof is a recursive function, and a recursive function is an inductive proof.

2.2 Addition

Let's understand his example for addition. 3 + 5 should be the same as (((5++)++)++) and adding 0 to 5 should just give 5.

Definition 2.2.1 notice he has a base case (0 + m is m), the hypotenuse "Suppose we inductively defined how to add n to m" which we just saw at the beginning of 2.2 where 3 + 5 is defined as incrementing five three times, finally the inductive step, we can add n++ to m by defining (n++) + m to be (n + m)++.

  • 4 + m = ((3++) + m) or ((((m++)++)++)++)
  • 2 + 3 = ((3++)++) or (4++) or 5.
  • 3 + 2 = (((2++)++)++) or ((3++)++) or (4++) or 5.

Lemma 2.2.2 notice first how he is using the exact same induction template from 2.1.11, you will eventually get very familiar with this template. Also notice how he's using prior established definitions to set up the proofs, notably Definition 2.2.1 where it was defined 0 + m = m. He assumed that n + 0 = n. Next the inductive step, where n++ + 0 = n++. No arithmetic was done here, he pointed to the definition of addition from 2.2.1 that n++ + 0 is (n + 0)++ or n++, so we have n++ = n++ and the proof is done.

He used a different variable to not confuse it with the previously established definition of 0 + m = m, since we are now going the other way with n + 0 = n on our way to prove a + b = b + a.

Lemma 2.2.3 Prove: n + (m++) = (n + m)++. We are inducting on n, and keeping m fixed which means we want to prove n++ not m++. Replace all instances of n with 0 for the base case and show the inductive hypothesis holds that 0 plus the increment of some other number, equals that number. Try writing this proof yourself to follow it:

  • base case:
    • 0 + (m++) = (0 + m)++
    • left side is m++ because of definition 2.2.1
    • right side is m++ because of definition 2.2.1
  • inductive hypotenuse n + m++ = (n + m)++
  • inductive step:
    • n++ + (m++) = (n++ + m)++
    • left side is ((n + m)++)++ because of 2.2.1
    • right side is ((n + m)++)++ because of 2.2.1

Exercise 2.2.1

Proposition 2.2.5 (Addition is associative) what we want to prove is (a + b) + c = a + (b + c). You don't have to be able to do these exercises, just try them and see if you can mimic what the author has been doing. Let's grab the inductive template and fill it in.

  • Base Case

    What is the base case? We want to show that even if a natural number is inside or outside brackets, it will have the same result when added together. We could pick b, then it would be (a + 0) + c = a + (0 + c) and from lemma 2.2.2/definition of addition, we would know this to be a + c = a + c but that doesn't seem to capture what we're trying to prove, and all of Tao's previous base cases we've seen tried to show what he was trying to prove not a trick to make both sides equal. Instead let's pick c or a, as on each side of the equality they are either inside the bracket, or outside it.

    (a + b) + 0 = a + (b + 0) (substitute 0 for c)

    • left side: (a + b) + 0
    • since b + 0 = b (lemma 2.2.2) then we have:
    • a + (b + 0) (lemma 2.2.2)
    • base case: a + (b + 0) = a + (b + 0)
    • or right side: a + (b + 0)
    • since b + 0 = b (lemma 2.2.2) then we have:
    • (a + b) + 0
    • another base case: (a + b) + 0 = (a + b) + 0
  • IH

    The inductive hypothesis we assume addition is associative, and that (a + b) + c = a + (b + c). Whenever you move around your proof in the inductive step, if it is either in form (a + b) + c or a + (b + c) you can change them for each other.

  • Inductive step

    There's a few ways to show this. I'm just going to move stuff around randomly and you can see where the proof is:

    (a + b) + (c++) = a + (b + (c++))

    • right side: a + (b + (c++))
    • a + (b + c)++ (lemma 2.2.3)
    • ((a + b) + c)++ (by the IH)
    • ((a + b)++) + c (def of addition)
    • ((b + a)++) + c (commutative)
    • ((b++)+ a) + c (def of addition)
    • c + ((b++) + a) (commutative)
    • c + (a + (b++)) (commutative)
    • left side: (a + b) + (c++)
    • (c++) + (a + b) (commutative)
    • (c + (a + b))++ (def of addition.. consider (a + b) to be one value
      • Recall (n++) + m = (n + m)++ ergo (n++) + (m + b) is (n + (m + b))++
    • ((a + b) + c)++ (commutative)
    • (a + (b + c))++ (by the IH)
    • ((b + c) + a)++ (commutative)
      • etc., can keep moving these around

    You should try this yourself, move everything around until you understand exactly how commutativity and associativity work. See if you can get it to a form where you can use the inductive hypothesis, permute them around some more until this starts to make sense. Remember to think as (n + m) as one value, so (n++) + (n + m) is (n + (n + m))++ and (n + (n + m)) can be rewritten according to commutative property ((n + m) + n).

    Proposition 2.2.6 (Cancellation Law) uses Axiom 2.4 which states 'Different natural numbers must have different successors; i.e., if n, m are natural numbers and n = m, then n++ = m++. Equivalently, if n++ = m++, then we must have n = m'. So (a + b)++ = (a + c)++ is = a + b = a + c by the line if n++ = m++, then n = m in axiom 2.4. We dropped down from the successor by using axiom 2.4 and can now just use our inductive hypothesis of the cancellation law for a to show b = c and close the induction. Notice how this is basically logical detective work, you look around for axioms and prior established lemmas and proofs to help you write a new proof.

    Lemma 2.2.10 has an exercise where induction is suggested. For the base case it's clear we can't use a, since (b++) = 0 would violate Axiom 2.3 that zero is not the successor of any natural number. So it must be b, (0++) = a, which by the definition of addition tells us (0++) = 1. The inductive hypothesis is that there exists exactly one natural number b, such that b++ = a, but this is already true by Axiom 2.4 where different natural numbers have different successors.
